\part{System Overview}

\section{Introduction}

Hydra is a containerized development platform providing persistent development environments for Computer Science students and faculty at SUNY New Paltz. The system runs on a 3-node RKE2 Kubernetes cluster with GPU acceleration for AI/ML workloads.

\subsection{Key Features}
\begin{itemize}
    \item \textbf{SSO Authentication:} Azure AD SAML 2.0 with automatic user provisioning
    \item \textbf{Persistent Containers:} Per-student environments with SSH, VS Code, Jupyter, and Jenkins
    \item \textbf{GPU Computing:} 5 GPUs across 2 nodes (3x RTX 3090 + 2x RTX 5090)
    \item \textbf{AI Chat:} OpenWebUI + KubeAI (vLLM models with scale-to-zero)
    \item \textbf{Dynamic Routing:} Traefik v2.11 reverse proxy with ForwardAuth
    \item \textbf{Workflow Automation:} n8n with integrated user management
    \item \textbf{21 TB Storage:} RAID-10 ZFS array with NFS exports for cross-node access
\end{itemize}

\subsection{Network Architecture}

\begin{itemize}
    \item All nodes on 192.168.1.0/24 LAN (gateway 192.168.1.1)
    \item Direct ethernet bridge between Chimera and Cerberus (reserved for RDMA/RoCE)
    \item WireGuard VPN: Chimera \texttt{wg0} = 10.8.0.2, Cerberus \texttt{wg0} = 10.8.0.3
    \item Canal CNI (Calico + Flannel VXLAN on port 8472/udp) for K8s pod networking
    \item UFW firewall on all nodes --- workers expose only SSH publicly
\end{itemize}

\section{Core Services (hydra-system)}

\subsection{Traefik (Reverse Proxy)}

\begin{infobox}
\textbf{Traefik v2.11} serves as the cluster ingress controller. It runs on Hydra with \texttt{hostPort} binding on ports 80 and 443. The deployment uses \texttt{strategy: Recreate} to avoid hostPort conflicts during rolling updates. Health probes on port 8082 (\texttt{--ping=true}) with PodDisruptionBudget (\texttt{minAvailable: 1}).
\end{infobox}

\textbf{Priority class:} \texttt{infra-critical} (value 1000000) applied to all infrastructure services. Student pods use default priority and are evicted first under memory pressure.

\textbf{Manifests:} \texttt{k8s/components/traefik/}

\subsection{Hydra Auth (SAML Gateway)}

The main authentication and container management service. Handles:
\begin{itemize}[nosep]

    \item JWT cookie issuance and JWKS endpoint
    \item Student container lifecycle (create, start, stop, destroy, wipe)
    \item Dashboard UI, admin panel
    \item OpenWebUI and n8n account provisioning
    \item WebSocket terminal bridge
\end{itemize}

\textbf{Manifests:} \texttt{k8s/components/hydra-auth/}

React frontend + Express backend + SQLite database (single pod). Serves the department homepage at \texttt{hydra.newpaltz.edu}.

\textbf{Database:} SQLite at \texttt{/app/server/data/cslab.db}. Persisted via PVC \texttt{cs-lab-data}.

\begin{enumerate}[nosep]

\section{Known Vulnerabilities}

\begin{dangerbox}
\textbf{High:}
\begin{itemize}[nosep]
    \item Passwordless sudo for student user inside containers
    \item Supervisor web interface (port 9001) without authentication (not exposed externally)
    \item K8s pod security set to Privileged at namespace level (lab environment tradeoff)
\end{itemize}

\textbf{Medium:}
\begin{itemize}[nosep]
    \item No NetworkPolicy isolation between student pods
    \item No PID limits (fork bomb vulnerability)
    \item Jupyter/VS Code without application-level auth (relies on ForwardAuth)
\end{itemize}
\end{dangerbox}

\section{Migration History}

The infrastructure evolved through several phases:
\begin{enumerate}[nosep]
    \item \textbf{Bare metal} (pre-2025) --- Apache web server, manual user management
    \item \textbf{Docker Compose} (2025) --- Containerized services, Nginx reverse proxy
    \item \textbf{K3s} (late 2025) --- Initial Kubernetes, migrated from Docker Compose
    \item \textbf{RKE2} (January 2026) --- Current production cluster, Traefik ingress
    \item \textbf{Infrastructure Overhaul} (February 9, 2026) --- OOM fix, Docker removal, data migration to ZFS
    \item \textbf{KubeAI Migration} (February 2026) --- Replaced Ollama with KubeAI/vLLM for scale-to-zero GPU inference
\end{enumerate}
